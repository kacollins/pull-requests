# Pull Requests and Merge Conflicts

## Creating a Repository from Visual Studio
1. Create a console app called `OpinionsFrom<YourFirstName>`
1. Go to the Git menu > Create Git Repository > Create and Push
1. Print some opinions to the console
    * These should be things that a couple of other people in the class will want to change.
	* They can be sports predictions, favorite food/drink, cutest animal, etc.
1. Save and run (F5) to test
1. Go to Git Changes to review your modified files
1. Stage the file you changed
1. Enter a commit message ("Add opinions") and commit the staged file
1. Push to GitHub

## Forking a Repository
1. Go to your profile on GitHub 
1. Find the list of users you’re following
1. Find someone’s opinions repo
1. Click the Fork button in the upper-right corner

## Creating a Branch
1. Clone Repository > GitHub button to browse
1. Select the other person’s opinions repo and click Clone
1. Create a branch called `update-to-opinions-from-<YourFirstName>`
1. Make a change, save it, and run it (F5) to test

## Creating a PR
1. Go to Git Changes to review your modified files
1. Stage the file you changed
1. Enter a commit message and commit the staged file
1. Push the branch to GitHub
1. Create pull request in Visual Studio
1. Click Create button
